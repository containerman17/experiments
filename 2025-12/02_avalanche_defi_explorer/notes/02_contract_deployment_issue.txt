# Task: Resolve Contract Deployment Regression

## Status: ✅ RESOLVED

## Root Cause
Scripts were using `localhost:9650` which pointed to a local testnet, while the actual node is at `167.235.8.126:9650`. The "old working contract" `0xcc57...` existed on the real network but not on the testnet, causing "no bytecode" errors.

## Fix Applied
1. Added `RPC_URL=http://167.235.8.126:9650/ext/bc/C/rpc` to `.env`
2. Created `pkg/rpc.ts` helper that throws if `RPC_URL` is not set
3. Updated all scripts to use `getRpcUrl()` from `pkg/rpc.ts` instead of hardcoded URLs

## Verified Working
- HelloWorld contract deployed and `greet()` returns "Hello, World!" ✅
- Hayabusa contract deployed at `0xd20bdfbed953a487871889211849a1de7e2db36c` ✅
- `owner()` call returns correct address ✅
- Bytecode length: 9944 bytes ✅
